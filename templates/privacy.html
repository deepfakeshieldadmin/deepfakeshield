Privacy Policy

Privacy & Data Handling

This page explains how uploaded files, scan results, and account data are handled within the DeepFake Shield system.

Uploaded Media

Uploaded files are used only for authenticity analysis and result generation. They may be stored in the media directory for scan history and PDF reporting.

Authentication Data

User accounts are managed through Django authentication. Passwords are hashed securely and are never stored in plain text.

Scan History

Scan results are stored for the logged-in user to provide historical access, result viewing, and PDF report downloads.

Project Limitation

DeepFake Shield provides practical authenticity estimation, not final forensic proof. Decisions should not rely only on the generated score.
